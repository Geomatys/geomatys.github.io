OGC TestBed-19

The OGC TestBed-19 proposes extensions to existing geospatial standards for handling data in space. At the most fundamental level, a few changes were proposed for the ISO 19111 standard. Those changes are then propagated to the OGC GeoAPI interfaces, the Apache SIS implementation, and the GeoTIFF tags. Since the TestBed is experimental, those changes were applied in branches. Links are given below, and their content may change at any time if future TestBeds continue this work or if some proposals are integrated in standards.

  • Source code

    • GeoAPI TestBed-19 branch — for new Coordinate Reference System type are related types.
    • Apache SIS TestBed-19 branch — for support of new GeoTIFF tags.

  • Compiled code (not an official release)

    • apache-sis-2.0-SNAPSHOT.zip — includes GeoAPI and Apache SIS branches.

  • Requirements (not included in above links)

    • Java 11 or later (Java 21 is recommended).
    • Optionally, for visualization purpose only: Open JFX.

For testing, unzip the apache-sis-2.0-SNAPSHOT.zip file in any directory. The instructions on the Apache SIS web site can be applied, with only the ZIP file changed. Then copy the TestBed-19 experiment data nearby. Those data are an image of Didymos asteroid and its Dimorphos moonlet seen by the DART before the impact, together with auxiliary files for referencing and metadata. The encoding experiment can be done by running the following commands in a Java shell (jshell). The paths to the Apache SIS directory and to the DART data directory may need to be adapted. 

./apache-sis-2.0-SNAPSHOT/bin/sis_shell

The following commands are Java code executed in the JShell, not bash commands. The first command encodes the DART_camera.png image in a new GeoTIFF file named DART_camera.tiff. The second command verifies the result. 

SIS.TRANSLATE.output("data/DART_camera.tiff").run("data/DART_camera.png")
SIS.CRS.run("data/DART_camera.tiff")
